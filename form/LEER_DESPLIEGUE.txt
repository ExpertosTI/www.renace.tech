## 1. COMANDO DE DESPLIEGUE RÁPIDO

Si ya subió la carpeta, copie y pegue esto en su terminal (asumiendo ruta `/var/www/renace.tech/form`):

```bash
cd /var/www/renace.tech/form && docker-compose down && docker-compose up -d --build && docker ps | grep renace
```
Esto:
1. Entra a la carpeta.
2. Apaga cualquier versión anterior.
3. Construye y levanta la nueva versión en puerto 8555.
4. Muestra si está corriendo.

---

## 2. CONFIGURAR NGINX (Para usar renace.tech/form)

Para que funcione en `renace.tech/form` sin poner puertos raros, debe editar la configuración de Nginx de su dominio `renace.tech`.

1. Abra su archivo de config (ej: `/etc/nginx/sites-available/renace.tech`).
2. Agregue este bloque dentro de `server { ... }`:

```nginx
location /form/ {
    # Redirige el tráfico al contenedor Docker en el puerto 8555
    proxy_pass http://localhost:8555/;
    
    # Cabeceras importantes
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

3. Guarde y reinicie Nginx:
```bash
sudo service nginx reload
```

## RESUMEN:
- El usuario entra a: `https://renace.tech/form`
- Nginx recibe y dice: "Ah, es /form, se lo paso al Docker en el 8555".
- Docker procesa el PHP y devuelve la respuesta.
- El usuario nunca ve el puerto 8555.
